# Blog Evolution Architecture

我们的目标不是复刻 Chenyme Blog。

我们的目标是：

构建一个：

- 长期维护
- 极高可定制性
- 极低耦合
- 完全自主可控
- 可无限扩展
- 极简依赖
- Personal OS

的网站。

原则：

UI 可以复杂。

数据必须简单。

架构必须简单。

依赖必须少。

优先级：

内容 > UI > SEO > 功能 > SaaS。

绝不为了功能引入服务。

只有真正需要时才引入服务。

所有新增依赖必须回答：

1. 是否必须？
2. 是否可替代？
3. 是否容易迁移？
4. 是否会造成 Vendor Lock？
5. 是否真的值得维护？

如果不能回答：

不引入
